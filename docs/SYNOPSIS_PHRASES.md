**5.2 Шаблоны под конкретные дизайны**
{% if design.recommended == "2x2" %}
Рекомендуется стандартный 2×2 кроссовер-дизайн как базовый вариант при отсутствии признаков высокой вариабельности или непрактичности washout.

{% elif design.recommended == "parallel" %}
Рекомендуется параллельный дизайн как альтернатива кроссоверу в ситуациях, где washout чрезмерно длителен, либо риск carry-over считается значимым.

{% if pk.t12_hours is defined and pk.t12_hours >= 24 %}

## 6) Статистический подход (короткая “рег-нейтральная” формулировка)
Биоэквивалентность оценивается на основании отношения геометрических средних значений (Test/Reference) PK-параметров с построением 90% доверительных интервалов после лог-трансформации соответствующих параметров. Критерии принятия и дополнительные условия (например, для NTI или высоковариабельных продуктов) определяются с учётом применимого регуляторного подхода и особенностей продукта.

## 9) Регуляторные аспекты и “точки внимания”
### 9.1 Стандартные пределы (общая формулировка)
Для большинства IR форм и стандартных сценариев BE критерии принятия определяются заранее установленными пределами для 90% CI отношения (T/R) по AUC и Cmax; применимость конкретных пределов зависит от юрисдикции и статуса продукта.

### 9.2 NTI (если применимо)
{% if drug.narrow_therapeutic_index %}
Препарат отмечен как NTI; следует рассмотреть применение ужесточённых пределов принятия для AUC и, при клинической значимости, для Cmax, а также отразить это в протоколе и планах статистического анализа.
{% endif %}

### 9.3 Высоковариабельные продукты (если применимо)
{% if cv.cvintra.value is defined and cv.cvintra.value >= 0.30 %}
Отмечается высокая вариабельность (CVw ≥ 30%): целесообразно рассмотреть replicate-дизайн. При необходимости для Cmax может обсуждаться альтернативный регуляторный подход, зависящий от CVw, с обязательным предварительным описанием в протоколе.
{% endif %}

### 9.4 Fed/Fasted (если требуется уточнение)
{% if design.fed_fasted == "unknown" %}
Требуется подтверждение режима (fasted/fed) на основании инструкции к референсному препарату и/или продукт-специфичных рекомендаций.
{% endif %}

### 9.5 CVfromCI (если применимо)
{% if cv.cvintra.source == "derived_from_ci" %}
Поскольку CVintra получен из 90% CI (CVfromCI), требуется явное документирование допущений: тип CI, анализ на log-шкале, дизайн/df, корректность n, соответствие условий (fasted/fed) и популяции.
{% endif %}
